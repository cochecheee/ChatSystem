Fix two issues on the Pipelines tab:

1. **Stale "No runs" on tab activation** — When the user navigates to the Pipelines tab, occasionally the page shows "No runs" even when runs exist server-side. Purpose: Reliable, scannable Pipelines view that distinguishes build/test pipelines from deploy pipelines and never shows stale "No runs" state.

Output: Updated `dashboard/src/pages/Pipelines.tsx` with consistent api helper usage, explicit refetch on mount with state reset, and a CI/CD-grouped sidebar.

   ```ts
   const CD_KEYWORDS = ['cd', 'deploy', 'release', 'publish', 'staging', 'production', 'prod'];

   function categorizeRun(run: WorkflowRun): 'ci' | 'cd' {
     const n = (run.name ?? '').toLowerCase();
     // word-ish match: split on non-alpha so 'cd' matches 'CD' but not 'codeql' or 'cdn'
     const tokens = n.split(/[^a-z0-9]+/).filter(Boolean);
     for (const kw of CD_KEYWORDS) {
       if (tokens.includes(kw)) return 'cd';
     }
     // fallback substring check for compound names like "deploy-prod"
     if (CD_KEYWORDS.some(kw => kw.length >= 4 && n.includes(kw))) return 'cd';
     return 'ci';
   }
   ```
   Rationale for tokenization: prevents false positives like "CodeQL" matching `cd` substring, while still catching compound names like `deploy-prod`.
