## Component Inventory

Components to build in Phase 4. All must follow existing CSS class conventions from index.css.

### New: DiffView Component

File: `dashboard/src/components/DiffView.tsx`

Structure:
```
.code-block (container)
  .code-block-header (file path + line count)
  <pre> (scroll container, overflow-x: auto)
    .diff-line* (one per line, grid: 36px 12px 1fr)
      .ln (line number, --fg-4, user-select: none)
      .marker (+ or -, centered, --fg-4 default)
      .content (line text, --font-mono, 12px)
    .diff-line.add (added lines)
    .diff-line.rem (removed lines)
```

Expand/collapse: lines > 20 — show first 10 and last 10 with a "Show N hidden lines" row between them. The expand row uses `.muted` style, font-size `--ts-xs`, centered, cursor pointer, background `--bg-muted`.

No external diff library. Parse `+`/`-`/`@@` prefixes inline. Source of truth: existing `.diff-line` CSS in index.css.

### New: PushPrButton (inline in Chat.tsx, not a standalone file)

Rendered below DiffView inside a `.msg-actions` container.

Elements:
- "Tạo PR" button — `.btn.primary` with "GitPullRequest" icon (16x16)
- "Hủy" button — `.btn.ghost` 
- PR link (after push completes) — rendered as inline anchor in `.msg-body`, accent color, opens in new tab

### Modified: Chat.tsx

Changes required:
1. Add `currentPatch` state: `{ findingId: number; patch: string; filePath: string } | null`
2. In `executeCommand`, when `cmd === 'fix'` and `res.data?.patch` present: set `currentPatch`, render DiffView in message bubble
3. Add `handlePushPR` handler: calls `/api/fix/{id}/push-pr`, shows PR URL in new `.msg.ai` message
4. Syntax hint for /fix: if `args[0]` is absent or NaN, show inline error "Cú pháp: /fix [finding_id]" (consistent with existing /approve guard in Chat.tsx line 154)
5. Suggestion chips update: add `/rerun [id]` to PRESETS list (currently missing from line 16)

### No Changes Required

- Shell.tsx — layout unchanged
- AlertBanner.tsx — already handles success/error/info types
- ApprovalDialog.tsx / RevokeDialog.tsx — already handle confirm/cancel pattern; PR push uses same inline button pattern, NOT a dialog

## Phase-Specific Constraints

1. No new npm packages. DiffView is implemented with inline parsing + existing `.diff-line` CSS classes — no diff library (consistent with RESEARCH.md "No New Packages to Install" finding).
2. No new CSS variables. All token references must resolve to existing `--` variables in tokens.css.
3. No modal for PR push approval. Use inline button pattern (`.msg-actions` row) — consistent with chat UX. Dialogs are reserved for /approve and /revoke only (existing ApprovalDialog / RevokeDialog).
4. DiffView uses existing `.code-block` + `.diff-line` CSS classes from index.css verbatim — no inline styles on the diff lines except for dynamic expand/collapse visibility.
5. MonoFont size for diff content is 12px (`--ts-sm`), NOT 11.5px — keep readable in constrained chat panel width (max 80% of panel on user side; full width for AI).
6. PR branch name displayed to user as code: wrap in `<code>` element (uses existing `.msg-body code` style: `--font-mono`, 11.5px, `--bg-muted` background, 3px border-radius).
