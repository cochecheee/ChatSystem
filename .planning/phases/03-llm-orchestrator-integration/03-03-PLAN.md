  <name>Task 1: ResponseValidator</name>
  <files>mcp/src/services/llm/validator.py</files>
  <action>
    - Tạo `mcp/src/services/llm/validator.py`.
    - Implement `ResponseValidator.clean(raw: str) -> str`:
      - Strip markdown code blocks nếu LLM bọc JSON trong ```json ... ```
      - Strip whitespace thừa đầu/cuối
    - Implement `ResponseValidator.validate(raw: str) -> AnalysisOutput`:
      - Gọi `clean()` trước
      - Parse với `AnalysisOutput.model_validate_json()`
      - Raise `LLMValidationError` nếu thất bại kèm error detail
    - Implement `ResponseValidator.fallback(finding_id: str) -> AnalysisOutput`:
      - Trả về AnalysisOutput hợp lệ với:
        - `explanation_vi = "Không thể phân tích tự động. Vui lòng xem xét thủ công."`
        - `confidence = "LOW"`
        - các field khác = giá trị placeholder an toàn
    - Định nghĩa custom exceptions:
      ```python
      class LLMValidationError(Exception): ...
      class LLMRateLimitError(Exception): ...
      class LLMServiceError(Exception): ...
      ```
  </action>
  <verify>
    <automated>pytest mcp/tests/services/llm/test_validator.py -v</automated>
  </verify>
  <done>Validator làm sạch JSON thừa markdown, raise đúng exception, fallback hợp lệ.</done>

<task type="auto">
  <name>Task 2: Tích hợp Validator và Retry vào Service</name>
  <files>mcp/src/services/llm/service.py</files>
  <action>
    - Cập nhật `LLMAnalysisService.analyze_vulnerability` để tích hợp validator:
      ```
      Lần 1: generate → validate → return nếu OK
      Lần 2 (nếu LLMValidationError): generate lại với error hint → validate → return nếu OK
      Lần 3 (nếu vẫn lỗi): return validator.fallback(finding_id)
      ```
    - Xử lý riêng từng loại exception:
      - `LLMRateLimitError` (429): exponential backoff, retry
      - `LLMValidationError`: retry với correction prompt
      - `LLMServiceError` (503/500): retry tối đa, sau đó fallback
    - Đảm bảo KHÔNG bao giờ raise exception ra ngoài service — luôn trả về `AnalysisOutput`.
  </action>
  <verify>
    <automated>pytest mcp/tests/services/llm/test_service.py -k "test_error_handling" -v</automated>
  </verify>
  <done>Service hoạt động bền bỉ: retry đúng loại lỗi, fallback khi cần.</done>
</task>
