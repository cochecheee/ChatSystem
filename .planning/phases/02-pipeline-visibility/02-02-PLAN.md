      if (!run.updated_at || !run.created_at) return '—';
      const ms = new Date(run.updated_at).getTime() - new Date(run.created_at).getTime();
      const s = Math.floor(ms / 1000);
      const m = Math.floor(s / 60);
      const h = Math.floor(m / 60);
      if (h > 0) return `${h}h ${m % 60}m`;

    }

| Boundary | Description |
|----------|-------------|
| GitHub API response → branch state | Branch names from GitHub runs populate the <select> options |
| User <select> input → filteredRuns filter | User-selected branch value is used to filter client-side array |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | XSS | Branch names rendered as `<option>` text content | accept | React JSX escapes all string content automatically; branch names originate from GitHub API (trusted source); no `dangerouslySetInnerHTML` |
| T-02-05 | XSS | Run names, tool names in RunSummaryStrip | accept | All rendered via JSX string interpolation — React escapes automatically |
| T-02-06 | Tampering | Branch filter client-side only — bypassing filter doesn't change backend data | accept | Filter is purely cosmetic UX — it does not grant access to data the user doesn't already have (all runs are fetched upfront) |
</threat_model>
