<feature>
  <name>Data Normalization Layer</name>
  <files>mcp/src/services/normalizer.py, mcp/tests/test_normalizer.py</files>
  <behavior>
    - **SARIF Support:** Parse SARIF 2.1.0 (Semgrep, CodeQL) and extract findings into unified `FindingCreate` schema.
    - **SpotBugs Adapter:** Parse SpotBugs XML and map rule IDs/severities to internal schema.
    - **ESLint Adapter:** Parse ESLint JSON format.
    - **Deduplication:** Filter out findings that match an existing hash (rule_id + file_path + fingerprint).
  </behavior>
  <implementation>
    - Implement `BaseNormalizer` (abstract class).
    - Implement `SarifNormalizer` using `sarif-pydantic`.
    - Implement `SpotBugsXMLNormalizer` using `xml.etree.ElementTree`.
    - Implement `NormalizerFactory` to select the correct normalizer based on file content/extension.
  </implementation>
</feature>

<feature>
  <name>Security Data Enrichment</name>
  <files>mcp/src/services/enricher.py, mcp/tests/test_enricher.py</files>
  <behavior>
    - **CWE Mapping:** Given a CWE ID, return the name and parent OWASP Top 10 category.
    - **CVSS Score:** Parse CVSS vectors or map severities (Low/Medium/High/Critical) to standard scores if vectors are missing.
    - **OWASP 2021 Mapping:** Categorize findings into the OWASP Top 10 list.
  </behavior>
  <implementation>
    - Use `cwe2` library for MITRE CWE database access.
    - Use `cvss` library for vector parsing.
    - Implement a mapping table for common SAST severities to CVSS scores.
  </implementation>
</feature>
